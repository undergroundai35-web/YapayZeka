1. Genel Bakış ve Teknoloji Yığını

Univera Connect (UniCP), Univera müşterilerinin finansal durumlarını, destek taleplerini, geliştirme isteklerini ve sistem performanslarını tek bir yerden izleyebilecekleri, B2B odaklı bir web portalıdır.

Teknoloji Stack'i

Katman Teknoloji Detaylar
Framework .NET 9.0 (ASP.NET Core MVC) High-performance, cross-platform backend.
Veritabanı (ORM) Entity Framework Core 9 Code-first ve DB-first hibrit yaklaşım (Stored Proc yoğunluklu).
Frontend Razor Views + Tailwind CSS Sunucu taraflı render (SSR). Tailwind CDN üzerinden yüklenir (Prod ortamında build pipeline önerilir).
UI Kütüphaneleri Bootstrap 5 (Legacy), Chart.js Eski bileşenler Bootstrap kullanırken, yeni modern arayüzler Tailwind tabanlıdır.
Servisler SmtpEmail, ZabbixService, AIService Dış sistemlerle haberleşme katmanı.

2. Sistem Mimarisi ve Entegrasyonlar

Proje Monolitik MVC mimarisine sahiptir ancak servis tabanlı bir yaklaşım izler. Veri erişimi ağırlıklı olarak Stored Procedure'ler üzerinden yürütülür, bu da iş mantığının bir kısmının veritabanında olduğu anlamına gelir.

Dış Sistem Entegrasyonları

  • TFS (Azure DevOps): Geliştirme talepleri SP_TFS_GELISTIRME prosedürü ile TFS veritabanından çekilir. Portal üzerinde yapılan durum güncellemeleri de TFS ile senkronize çalışır.
  • ERP (Logo/Netsis): Fatura ve sipariş verileri SP_VARUNA_SIPARIS ile ERP sisteminden beslenen ara tablolardan okunur.
  • Zabbix: Sunucu performans verileri (CPU, RAM, Disk) Zabbix API üzerinden JSON-RPC protokolü ile çekilir. (Bkz: ZabbixService.cs).
  • AI (Gemini/Ollama): Proje içinde yapay zeka destekli analizler için entegrasyon altyapısı mevcuttur.

3. Kimlik Doğrulama ve Rol Yönetimi

Sistem, ASP.NET Core Identity üzerine inşa edilmiş ancak özelleştirilmiş bir claim (hak) yapısı kullanır.

Kullanıcı Tipleri (UserType Claim)

Kullanıcı giriş yaptığında CustomUserClaimsPrincipalFactory devreye girer ve TBL_KULLANICI tablosundan kullanıcının tipini okuyarak UserType claim'i oluşturur.

Tip ID Rol Adı Badge Tanım
1 Admin Yönetici Tüm firmalara ve tüm verilere erişebilir. Global filtreleme yapabilir.
2 Müşteri Müşteri Sadece kendi firmasına (veya atandığı projelere) ait verileri görür. Filtreleme kısıtlıdır.
3 Univera Personel Univera iç çalışanlarıdır. Birden fazla firmaya destek verebilirler. Yetkili oldukları firmalar arasında geçiş yapabilirler.
Önemli: View tarafında yetki kontrolleri genellikle User.FindFirst("UserType") veya User.IsInRole("Admin") ile yapılır.

4. Veritabanı ve Kritik Tablolar

Proje iki ana DbContext kullanır: DataContext (Identity tabloları için) ve MskDbContext (İş verileri için).

Kritik Tablolar

  • TBL_KULLANICI: Kullanıcı profilleri, tipleri ve firma kodları.
  • TBL_KULLANICI_FIRMA: Univera (Tip 3) kullanıcılarının hangi firmalara bakabileceğini tutan eşleşme tablosu.
  • TBL_VARUNA_SOZLESME: Lisans ve sözleşme başlıkları.
  • VIEW_ORTAK_PROJE_ISIMLERI: Sistemdeki tüm müşteri/proje isimlerinin listesi. (Şirket dropdown'ı burayı kullanır).
  • TBL_TALEP: Portal üzerinden açılan veya TFS'den gelen taleplerin portal tarafındaki takibi.
  • TBL_TALEP_AKIS_LOG: Taleplerin durum geçmişini tutar (Analiz -> Bütçe -> Geliştirme logları).
  • TBL_FINANS_ONAY: Müşterinin sipariş onay/ret durumlarını tutar.

5. Modül Detayları

A. Finans Modülü (FinansController)

Müşterilerin bekleyen siparişlerini, faturalarını ve bakiyelerini gösterir.

  • Paralel İşleme: Admin veya Univera kullanıcıları için, çoklu firma sorguları Parallel.ForEachAsync ile optimize edilmiştir.
  • Onay Mekanizması: Müşteri "Onayla" dediğinde TBL_FINANS_ONAY tablosuna kayıt atılır ve arayüzde sipariş "Onaylandı" olarak görünür.

B. Geliştirme Talepleri (TaleplerController)

En karmaşık iş akışına sahip modüldür. TFS (DevOps) ile entegre çalışır. Müşteri buradan yeni geliştirme ister, bütçe onaylar veya UAT (Kullanıcı Testi) yapar.

C. Sistem Performansı (SystemPerformanceController)

Zabbix API'den sunucu sağlık verilerini çeker. RAM, CPU ve Disk kullanımı anlık olarak gösterilir.

6. İş Akışları (Workflows)

Geliştirme Talebi Süreci (Status Flow)

Bir talep açıldığında aşağıdaki yaşam döngüsünden geçer. Her aşamada görünen butonlar kullanıcı tipine göre değişir:

  1. 1. Analiz
    Talep açıldı, analiz yapılıyor.
    ➔ Univera: Analizi Tamamla butonunu görür. (Hedef: Analiz Onayı)
  2. 2. Analiz Onayı YENİ
    Analiz bitti, Univera içi onay/kontrol aşaması.
    ➔ Univera: Analizi Onayla butonunu görür. (Hedef: Bütçe Onayı)
  3. 3. Bütçe Onayı
    Müşteriye maliyet sunuldu.
    ➔ Müşteri: Bütçeyi Onayla veya Reddet butonlarını görür.
  4. 4. Geliştirme
    Kodlama yapılıyor.
    ➔ Univera: Geliştirmeyi Bitir butonunu görür. (Hedef: Proje Testi)
  5. 5. Proje Testi
    İç testler yapılıyor.
    ➔ Univera: UAT'e Gönder (Başarılı) veya Test Başarısız (Geri Dön) butonlarını görür.
  6. 6. Müşteri UAT
    Müşteri kendi testini yapıyor.
    ➔ Müşteri: Canlıya Al (Onay) veya Revize İste (Hata var) butonlarını görür.
  7. 7. Canlıya Geçiş (Production)
    Talep tamamlandı.

7. Kurulum ve Deployment

Gereksinimler

  • Windows Server / IIS
  • SQL Server (Veritabanı erişimi)
  • .NET 9 Hosting Bundle

IIS Kurulumu

  1. Projeyi dotnet publish -c Release ile derleyin.
  2. IIS'de yeni bir Website oluşturun.
  3. Application Pool ayarlarından ".NET CLR Version" kısmını "No Managed Code" olarak seçin (Kestrel kullanıldığı için).
  4. appsettings.json dosyasındaki DB Connection String'leri ve Zabbix IP'lerini sunucuya göre düzenleyin.
